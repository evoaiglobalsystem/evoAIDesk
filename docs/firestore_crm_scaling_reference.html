Important Findings

What Scales Well

  • Cursor-based pagination using limit() and startAfter()
  • Tenant isolation with tenant_id filtering
  • Small page sizes (15 docs)
  • Firestore count aggregation queries
  • Soft-delete filtering

Primary Bottlenecks

  • Client-side search filtering
  • Large kanban board stage loading
  • Realtime listeners on dashboards
  • Repeated analytics refreshes
  • Large multi-stage board queries
